Developers collect failure reports from software applications in operation to debug the origins of faults in software code. Crash reports of applications with large client bases can be quite overwhleming to software developers, let alone the complexity of faults to be debugged. Manual identification of faulty code from these failure reports is an arduous task and can consume upto 40-90% of the total time needed to fix the fault. Debug007 is a tool that automatically pinpoints the locations of faults in the failure reports of the deployed software applications. Initially, Debug007 will be available as SaaS for mobile applications.

Studies show that approximately 50-90% of failures in deployed applications occur due to previous faults and a majority of them originate from only 20% of the code. Using machine learning algorithms, Debug007 exploits this behaviour of software applications to point out a list of suspected functions or components for a failure report. Our case studies show that developers can diangose faulty source code for failure reports by reviewing first few functions/components from the suspected list. However, contemporary commerical technologies avialable to debug failure reports only provide a set of different views which developers need to correlate manually to diagnose the fault.

FAQ

Who are we?
Debug007 is a product of research conducted on software behaviour at Concordia University and Western University of Canada. Several case studies have been performed on commerical and public programs about the validity of Debug007 technique. Results show that faults in 70-90% of failure reports can be found by reivewing upto top 3 functions from the suspected list of Debug007.
